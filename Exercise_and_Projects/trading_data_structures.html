Comparison

Side-by-side summary of when to use each structure

List

[ ]

Use when you need an ordered, changeable collection. Best for watchlists, price series, or trade logs that will be modified.

✓ add / remove items
✓ sort and reverse
✓ mixed types ok
✓ duplicates ok
✗ not hashable

Tuple

( )

Use for fixed data that should not change. OHLC bars, trade records, (ticker, exchange) pairs as dict keys.

✓ faster than list
✓ hashable (dict key)
✓ safe / read-only
✓ unpack easily
✗ cannot modify

Dictionary

{ }

Use when you need named fields, fast lookups by key, or a mapping between concepts portfolios, stock quotes, order books.

✓ O(1) key lookup
✓ named fields
✓ mutable values
✓ nesting possible
✗ no duplicate keys

Memory and performance (approximate)

List - moderate overhead~56 + 8n bytes
Tuple - smallest~40 + 8n bytes
Dict - largest~240 + ~50n bytes
